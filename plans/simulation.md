# 시뮬레이션 기능 설계 (Phase 5)

> 2026-06-07 조사. 디지털 트윈은 실제 도시가 아니므로 "만약 ~라면" 시나리오를 보여주는 시뮬레이션이 핵심 차별 기능.
> 데이터 출처는 [data/SOURCES.md](../data/SOURCES.md) 기준.
> **✅ 채택 결정: C안 (전체 5종 — 그림자·침수·CCTV커버리지·교통우회·대피)** — 2026-06-07 사용자 결정.
> 구현 순서는 난이도 낮은 것부터: 1 → 2 → 3 → 4 → 5.

## 후보 시뮬레이션 유형

### 1. 일조/그림자 시뮬레이션 — 난이도 하 ★ 가성비 최고

- **내용**: 시간 슬라이더로 태양 위치를 바꾸며 건물 그림자 변화를 애니메이션
- **필요 데이터**: 없음 (Cesium 내장 `viewer.shadows` + OSM Buildings / GIS건물통합정보 3D)
- **구현**: 순수 클라이언트. `viewer.clock` 시간 조작 + shadows 활성화
- **비고**: 서울 S-Map·브이월드 디지털트윈의 단골 기능. 구현 비용 대비 시연 효과 큼

### 2. 침수 시뮬레이션 (bathtub 모델) — 난이도 하~중 ★ 추천

- **내용**: "강우량/수위 H" 슬라이더 → 고도 < H 영역을 반투명 파란색으로 표시
- **필요 데이터**: DEM(국토정보플랫폼 5m 또는 NSDI 90m), 검증용 침수흔적도(safemap WMS), 강우 연동 시 기상청 초단기실황 API
- **구현**:
  - 백엔드: PostGIS 래스터로 DEM 적재 → 임계값 폴리곤화(`ST_DumpAsPolygons`) → GeoJSON 응답
  - 또는 클라이언트: Cesium 수면 폴리곤 높이 조절 (간단 버전)
- **비고**: 정밀 유체 모델(HEC-RAS)은 범위 밖. "수위 채우기" 방식이 졸업작품 수준에서 표준적

### 3. CCTV 감시 커버리지 — 난이도 하(2D)~상(3D)

- **내용**: CCTV 위치 기준 감시 범위 표시 + 감시 공백 지역 도출 (Phase 4 위기레벨과 연계)
- **필요 데이터**: 보유한 방범CCTV 21,060건 (장비종류 컬럼 활용 가능), 3D 정밀화 시 건물·DEM
- **구현**:
  - 2D: PostGIS `ST_Buffer` 반경 + 화각 부채꼴 — 즉시 가능
  - 3D viewshed: 건물 차폐 고려 가시영역 — 가산점 요소 (GRASS `r.viewshed` 오프라인 계산 후 적재)
- **비고**: 위기레벨 산식(4-1)의 CCTV 커버리지 요소를 시각화로 확장하는 셈이라 시너지 큼

### 4. 교통 시뮬레이션 (도로 차단 → 우회 경로) — 난이도 중

- **내용**: 지도에서 도로(링크) 클릭해 차단 → 차단 전/후 경로 변화 비교 표시
- **필요 데이터**: 국가표준노드링크(SHP, 부산 추출), 실측 연동 시 링크소통정보 API(링크별 속도)
- **구현**: PostGIS **pgRouting** `pgr_dijkstra` — 차단 링크 제외 후 재계산. 미시적 교통류(SUMO)는 범위 밖
- **비고**: "디지털 트윈다운" 인터랙션. 노드링크 적재·정제 작업이 선행 필요

### 5. 대피 시뮬레이션 (재난 → 최근접 대피소) — 난이도 중~상

- **내용**: 지진해일 등 재난 시나리오 → 격자 인구별 최근접 대피장소 경로/도달시간 표시
- **필요 데이터**: 지진해일 긴급대피장소 표준데이터(수용인원·해발고도 포함), 표준노드링크, SGIS 격자인구
- **구현**: pgRouting 최단경로 + 도달권(isochrone). 4번과 같은 기반 위에 올라감
- **비고**: 침수(2번)와 결합하면 "해일 수위 → 침수 구역 → 대피 경로" 스토리 완성

### 6. 기타 (참고만)

- 소음 확산(거리 감쇠 모델), 미세먼지 IDW 보간 히트맵(에어코리아 API) — 채택 시뮬레이션 확정 후 여력 있으면 검토

## 국내 사례 참고 (조사 메모)

- **브이월드 디지털트윈국토**(국토부), **서울 S-Map**: 일조·조망, 침수, 바람길 등 제공 — 일조/침수/가시권이 단골 기능
- 부산 스마트시티 국가시범도시(에코델타시티)도 데이터 기반 시뮬레이션 지향
- ※ 사례 기능 목록은 2차 자료 기반, 세부 검증 안 됨

### C안 채택에 따른 후속 사항 (1-R 테이블 재설계에 반영)

- 시뮬레이션 입력은 **facility와 별도 테이블**로 설계:
  - `building` — GIS건물통합정보 (폴리곤+층수, 그림자·차폐용)
  - DEM 래스터 — PostGIS raster 또는 사전 폴리곤화 결정 필요 (침수용)
  - `road_node` / `road_link` — 표준노드링크 부산 추출 (pgRouting 토폴로지, 교통·대피용)
  - `shelter` — 지진해일 긴급대피장소 (수용인원·해발고도 포함)
  - `population_grid` — SGIS 격자인구 (대피 수요)
- 필요 확장: PostGIS raster + **pgRouting** 익스텐션 설치 확인
- 추가 수집 필요 데이터: DEM, GIS건물통합정보(부산), 표준노드링크, 대피장소 표준데이터, SGIS API 키
