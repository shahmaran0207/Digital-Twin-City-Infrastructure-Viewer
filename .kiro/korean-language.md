# 한국어 응답 규칙

## 언어 설정
- 모든 응답은 한국어로 작성해야 합니다  
- 코드 주석도 가능한 한 한국어로 작성합니다  
- 기술 용어는 필요시 영어와 한국어를 병행 표기합니다 (예: "컨테이너(container)")  
- 에러 메시지나 로그는 원본 언어를 유지하되, 설명은 한국어로 제공합니다  

## 예외 상황
- 코드 자체는 영어로 작성 (변수명, 함수명 등)  
- 공식 문서나 명령어는 원본 언어 유지  
- 사용자가 명시적으로 다른 언어를 요청하는 경우에만 예외 적용  

## 주석 유지 원칙
- 기존 주석은 가능한 한 유지해야 합니다.  
- 해당 주석에 대응되는 코드가 삭제되어 의미가 사라진 경우에만 예외적으로 제거할 수 있습니다.  
- 주석이 여전히 문맥상 의미를 가지면 코드가 변경되더라도 보존합니다.

## 코드 작성 원칙
- 코드는 최대한 직관적으로 작성해야 합니다.

## 커밋 원칙
- 기능이 완성될때 마다 깃에 커밋을 해야 합니다.
- 테스트가 통과한 상태에서만 커밋합니다 (테스트가 아직 없는 영역은 빌드/기동 확인으로 대체).

## 테스트 원칙
- 계획(PLAN.md)의 한 단계가 끝날 때마다 **전체 테스트를 실행**해 통과를 확인한 뒤 다음 단계로 진행합니다.
  - 백엔드: `gradlew build` (컴파일 + 테스트 포함)
  - 프론트엔드: ESLint + `tsc -b` + `vite build`
- 테스트가 실패하면 다음 단계로 넘어가지 않고 원인을 먼저 해결합니다.
- 테스트로 확인할 수 없는 항목(지도 렌더링 등)은 실행 화면 확인으로 대체하고, 확인한 방법을 보고합니다.
